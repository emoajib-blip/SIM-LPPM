# Plan: Fix Scheme Eligibility Alert for Users with Existing Drafts

## Problem Statement
The system shows "Status Eligibilitas Skema: Tidak Memenuhi Syarat" (Scheme Eligibility Status: Does Not Meet Requirements) even when users have existing draft proposals for research and community service. This occurs because eligibility is checked in real-time against current SINTA scores and functional positions, discouraging users who created valid drafts when eligible.

The current logic:
- Scheme eligibility is strictly enforced based on current user data
- Drafts can be submitted if general lecturer obligations are met (no pending reports from previous semesters)
- Alert shows "not eligible for any schemes" if no schemes pass current checks, even if user has valid drafts

## Proposed Solution
Keep scheme eligibility strict as configured by admin (requiring SINTA scores, functional positions, etc.), but modify the UI alert to not display the discouraging message for users who have existing submittable proposals (DRAFT, NEED_ASSIGNMENT, REVISION_NEEDED). This allows users to submit their existing drafts while maintaining strict eligibility for new proposals.

## Implementation Steps

### 1. Revert Synchronization Changes
- Revert changes to EligibilityService.getEligibleResearchSchemes()
- Revert changes to EligibilityService.getEligibleCommunityServiceSchemes()
- Revert changes to LecturerEligibilityService.getScheduleStatus()

### 2. Update UI Alert Logic
- Modify research/proposal/index.blade.php to check if user has submittable proposals
- Modify community-service/proposal/index.blade.php similarly
- If user has submittable proposals, don't show the "not eligible for schemes" alert
- Keep scheme eligibility strict for new proposal creation

### 3. Test the changes
- Verify scheme eligibility remains strict for new proposals
- Confirm users with drafts don't see the eligibility alert
- Ensure submission works for existing drafts (already works since submission only checks general obligations)

## Benefits
- Maintains strict scheme eligibility as configured by admin
- Prevents user confusion for those with valid existing drafts
- No changes to core eligibility logic
- Simple UI-only fix

## Potential Concerns
- Users might still be confused why they can't create new proposals in certain schemes
- Alert is hidden but eligibility rules still apply

## Alternatives Considered
- Synchronize eligibility with existing drafts (too permissive, bypasses admin rules)
- Remove eligibility checks entirely (not secure)
- Show different alert text for users with drafts

## Files to Modify
- `resources/views/livewire/research/proposal/index.blade.php`
- `resources/views/livewire/community-service/proposal/index.blade.php`
