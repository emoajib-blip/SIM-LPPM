# Plan: Update Scheme Eligibility Logic to Synchronize with Existing Draft Proposals

## Problem Statement
The system shows "Status Eligibilitas Skema: Tidak Memenuhi Syarat" (Scheme Eligibility Status: Does Not Meet Requirements) even when users have existing draft proposals for research and community service. This occurs because eligibility is checked in real-time against current SINTA scores and functional positions, but users who created drafts when eligible become ineligible if their scores/positions change or scheme requirements are updated.

The current logic:
- Eligibility is checked dynamically for each scheme based on user identity data
- Draft creation allows proposals in ineligible schemes (only general lecturer obligations are checked)
- Submission requires both general eligibility AND scheme-specific eligibility
- Alert shows "not eligible for any schemes" if no schemes pass current checks, even if user has drafts

## Proposed Solution
Modify the scheme eligibility logic to include schemes where the user has existing submittable proposals (DRAFT, NEED_ASSIGNMENT, REVISION_NEEDED statuses). This "synchronizes" eligibility with the system state - if you have drafts, those schemes remain eligible for you.

## Implementation Steps

### 1. Update EligibilityService.getEligibleResearchSchemes()
- Add logic to include schemes where user has proposals in submittable statuses
- Use Eloquent query to find user's proposals with status in [DRAFT, NEED_ASSIGNMENT, REVISION_NEEDED] and research_scheme_id not null
- Merge these schemes into the eligible list

### 2. Update EligibilityService.getEligibleCommunityServiceSchemes()
- Same logic as above, but for community_service_scheme_id

### 3. Update EligibilityService.getResearchSchemeEligibilityBreakdown() and related methods
- Ensure breakdown methods account for the synchronized eligibility

### 4. Test the changes
- Verify that users with drafts see the schemes as eligible
- Confirm submission works for existing drafts
- Check that new proposals still require current eligibility

## Benefits
- Users can submit existing drafts even if their SINTA/position no longer meets current requirements
- Reduces user confusion and support requests to Admin LPPM
- Maintains security by still checking eligibility for new proposals
- Aligns with system logic where drafts represent prior approval

## Potential Concerns
- Users might game the system by creating drafts when eligible then updating their data
- Admin changes to scheme requirements might not immediately block existing drafts
- Need to ensure this doesn't bypass security checks

## Alternatives Considered
- Remove eligibility checks for submission of existing drafts (too permissive)
- Lock eligibility at draft creation time (complex state management)
- Show different UI state for "conditionally eligible" schemes (UI complexity)

## Files to Modify
- `app/Services/EligibilityService.php` - Core eligibility logic
- Potentially `app/Livewire/Research/Proposal/Index.php` and community service equivalent - If UI needs updates

## Testing Strategy
- Create test user with low SINTA score
- Have them create research and community service drafts when "eligible"
- Update their SINTA score to below minimum
- Verify schemes still appear eligible and submission works
