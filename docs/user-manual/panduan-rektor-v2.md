## 1. Pendahuluan

### 1.1 Peran Rector dalam SIM LPPM

Sebagai Rector, Anda memiliki peran pengawasan strategis tertinggi terhadap seluruh aktivitas penelitian dan pengabdian masyarakat di ITSNU Pekalongan. Sistem ini memberikan Anda pandangan makro (*bird's eye view*) tentang produktivitas akademik universitas.

### 1.2 Hak Akses Utama

| Fitur | Akses |
|-------|-------|
| Dashboard Eksekutif | ✓ |
| Semua Laporan | ✓ |
| Dashboard IKU | ✓ |
| Monitoring Seluruh Proposal | ✓ |
| Audit Log | ✓ |
| Persetujuan Strategis | ✓ |

### 1.3 Perbedaan dengan Kepala LPPM

| Aspek | Rector | Kepala LPPM |
|-------|--------|-------------|
| Fokus | Pengawasan strategis | Operasional LPPM |
| Keputusan | Monitoring saja | Keputusan final proposal |
| Akses data | Seluruh universitas | LPPM scope |

## 4. Monitoring IKU

### 4.1 Mengakses IKU

1. Klik **IKU** di sidebar
2. Dashboard IKU akan terbuka

### 4.2 Kategori IKU

| Kategori | Indikator |
|----------|-----------|
| **Publikasi** | Jurnal Nasional, Internasional |
| **HKI** | Paten, Hak Cipta, Merek |
| **Pengmas** | Jumlah masyarakat terdampak |
| **Kerjasama** | MoU/kerjasama baru |
| **Pendidikan** | Tingkat penyelesaian |

### 4.3 Visualisasi

Dashboard menampilkan:

1. **Progress Bar**: Target vs Realisasi per indikator
2. **Pie Chart**: Distribusi luaran
3. **Trend Line**: Perkembangan per tahun

### 4.4 Detail per Indikator

Klik pada indikator untuk melihat detail:
- Target tahunan
- Realisasi saat ini
- Gap yang perlu dikejar
- Daftar luaran

### 4.5 Verifikasi IKU

1. Klik **IKU** > **Verifikasi**
2. Review data luaran
3. Verifikasi atau tolak

- Pengusul

- Luaran

## 8. Persetujuan Strategis

### 8.1 Akses Persetujuan

Meskipun fokus monitoring, Rector dapat:

1. Jika ada proposal perlu persetujuan:

Rector dapat menyetujui:
- Laporan akhir penelitian
- Laporan monev

## 10. Troubleshooting

### 10.1 Masalah Umum

| Masalah | Solusi |
|---------|--------|
| Data tidak muncul | Cek filter sudah benar |
| Grafik tidak muncul | Refresh halaman |
| Export gagal | Cek koneksi internet |
| Notifikasi tidak masuk | Cek pengaturan notifikasi |

### 10.2 Pertanyaan Umum

**Q: Apakah saya bisa mengubah keputusan Kepala LPPM?**
A: Secara teknis ya, tetapi biasanya dilakukan secara koordinasi.

**Q: Berapa tahun data yang tersedia?**
A: Sejak sistem diimplementasikan.

**Q: Bisakah saya melihat detail proposal?**
A: Ya, klik pada baris proposal.

## Tips Penggunaan untuk Rector

### Penggunaan Dashboard

1. **Checklist Harian**:
   - Buka dashboard setiap pagi
   - Cek notifikasi penting
   - Review aktivitas terbaru

2. **Weekly Report**:
   - Eksport laporan mingguan
   - Review progress IKU
   - Identifikasi masalah

3. **Monthly Analysis**:
   - Analisis tren bulanan
   - Bandingkan dengan target
   - Rapat dengan Kepala LPPM
